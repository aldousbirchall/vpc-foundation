# vpc-foundation

Terraform configuration for a two-tier VPC foundation in AWS eu-west-1. Provides networking (public/private subnets across 2 AZs, NAT gateway, internet gateway), security groups for web and application tiers, and an S3 bucket for static assets.

> **This is a proof-of-concept infrastructure build** generated autonomously by [Software Factory](https://github.com/aldousbirchall/softwarefactory) to validate the factory's Terraform pipeline (v0.3.0). The configuration is structurally valid and has passed plan-level verification, but has not been applied to a live AWS account. It is a foundation layer only; no compute resources are provisioned.

## Architecture

```
                    Internet
                       |
                 [Internet Gateway]
                       |
          +------------+------------+
          |                         |
  [Public Subnet 1]        [Public Subnet 2]
   10.0.1.0/24              10.0.2.0/24
   eu-west-1a               eu-west-1b
          |
     [NAT Gateway]
     (Elastic IP)
          |
          +------------+------------+
          |                         |
  [Private Subnet 1]       [Private Subnet 2]
   10.0.10.0/24             10.0.20.0/24
   eu-west-1a               eu-west-1b
```

Single NAT gateway (single-AZ) for cost control. Estimated baseline: ~$33-38/month.

## Modules

| Module | Resources |
|---|---|
| `networking` | VPC, 4 subnets, internet gateway, NAT gateway, EIP, 2 route tables, 4 associations |
| `security` | Web SG (HTTP/HTTPS from 0.0.0.0/0), App SG (ingress from web SG only) |
| `storage` | S3 bucket with versioning, AES256 encryption, public access blocked |

## Usage

```bash
cd src
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars with your values (owner, bucket_name)

make init       # terraform init
make validate   # terraform validate
make plan       # terraform plan
```

No `apply` or `destroy` targets exist. This is a plan-only configuration.
